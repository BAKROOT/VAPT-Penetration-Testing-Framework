# Standards Mapping

This document maps our internal VAPT phases to widely recognized testing standards so that engagements remain defensible and aligned with industry expectations.

## Phase-to-Standard Mapping

| Our Phase | PTES | NIST SP 800-115 | OWASP / OSSTMM |
| --- | --- | --- | --- |
| Pre-engagement & Scoping | Pre-engagement Interactions | Planning | OWASP WSTG: Pre-engagement; OSSTMM: Scope |
| Reconnaissance | Intelligence Gathering | Discovery | OWASP WSTG: Information Gathering |
| Scanning & Enumeration | Intelligence Gathering / Threat Modeling | Discovery | OSSTMM: Enumeration |
| Vulnerability Analysis | Vulnerability Analysis | Discovery / Attack | OWASP WSTG test categories |
| Exploitation | Exploitation | Attack | OWASP WSTG validation |
| Post-exploitation | Post Exploitation | Attack | OSSTMM: Access |
| Reporting | Reporting | Reporting | OWASP reporting guidance |
| Remediation & Retest | (extension) | Post-test | Verification / retest |

## Standards Reference

- **PTES** — Penetration Testing Execution Standard: defines a baseline for what a penetration test should include.
- **NIST SP 800-115** — Technical Guide to Information Security Testing and Assessment: planning, discovery, attack, and reporting.
- **OWASP WSTG** — Web Security Testing Guide: detailed web application test cases.
- **OWASP API Security Top 10** — common API risks.
- **OWASP MASVS / MASTG** — mobile application security verification and testing.
- **OSSTMM** — Open Source Security Testing Methodology Manual: operational security testing.
- **CVSS v3.1** — severity scoring used across all findings.

## Why Mapping Matters

Mapping our process to established standards provides consistency, supports client compliance requirements, and ensures coverage is complete and defensible during quality review.
