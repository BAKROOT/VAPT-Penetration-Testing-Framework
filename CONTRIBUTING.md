# Contributing Guidelines

Thank you for helping improve the VAPT Penetration Testing Framework. This repository contains methodology, checklists, templates, and documentation for **authorized** security testing.

## Ground Rules

- **Authorized testing only.** Nothing in this repository should be used against systems without explicit written permission.
- **Never commit secrets, credentials, personal data, real client names, or live findings/evidence to this repository.** Sanitize all examples.
- Keep contributions vendor-neutral and standards-aligned (OWASP, PTES, NIST SP 800-115, OSSTMM, CVSS).
- Use clear, professional language suitable for client deliverables.

## How to Contribute

1. Fork the repository and create a feature branch (`feature/short-description`).
2. Make focused changes — one topic per pull request.
3. Follow the existing folder structure and naming conventions (lowercase, hyphen-separated file names).
4. Update any relevant README or index when you add files.
5. Open a pull request describing what changed and why.

## Style Conventions

- Markdown for documentation, CSV for registers and trackers.
- File names: `lowercase-with-hyphens.md`.
- Reference standards by name and section where relevant.
- Severity ratings must use CVSS v3.1 and map to a qualitative band (Critical/High/Medium/Low/Info).

## Reporting Issues

Open a GitHub issue for errors, gaps, or suggested additions. Do not include any real or sensitive data in issues.

## Code of Conduct

Be respectful and constructive. Security testing is a profession built on trust, integrity, and responsible disclosure.
