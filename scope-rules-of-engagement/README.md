# Scope & Rules of Engagement

This section defines how engagements are scoped and authorized. **No testing may begin until the Rules of Engagement (ROE) and authorization are signed by an authorized representative of the target organization.**

## Contents

| File | Description |
| --- | --- |
| `rules-of-engagement-template.md` | The ROE document to be completed and signed per engagement |
| `authorization-to-test.md` | Formal written authorization / "get out of jail" letter template |
| `scope-definition-template.md` | In-scope / out-of-scope asset definition |

## Why This Matters

Unauthorized testing is illegal and unethical. The artifacts here ensure that every engagement has explicit, documented, time-bounded permission, clear boundaries, and defined emergency procedures before any technical work starts.

## Golden Rules

- Never test anything not explicitly listed as in-scope.
- Stop immediately if you encounter systems, data, or impact outside the agreed scope, and notify the point of contact.
- Keep the signed authorization available throughout the engagement.
